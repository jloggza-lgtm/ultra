03

Optimization Roadmap

Track A
Smart Contract
4–6 weeks
Internal Ledger
Internal balance accounting removes token transfer overhead from the matching hot path, delivering approximately 2× TPS uplift across all orderbook configurations.
Table Restructuring
Orderbook table restructuring eliminates the state serialization penalty at depth — bringing large orderbook performance in line with small. Uniform throughput regardless of market depth.
Combined estimated outcome: 14,000+ TPS & 0.07 ms execution time on any sized orderbook (excluding network latency). A large orderbook is no longer a bottleneck.
Track B
Network Layer
TBD
Hardware Upgrades
Upgrading block producer hardware directly reduces computation time. Higher clock speed and more performant CPUs translate linearly into lower execution latency at the protocol level.
Network Proximity
Geolocalising block producers into a single region (EU), or deploying dedicated trading terminals co-located at each BP — both approaches eliminate geographic latency.
Private Flow
Dedicated API endpoint with direct connection to the producing block producer, bypassing the standard three-layer propagation path. Order flow is invisible to the network until block inclusion.
Transaction Queue Management
Under high volatility or extended queue build-up, a dedicated service can cancel queued but not-yet-included transactions, preventing stale orders from executing at bad prices.
Estimated outcome: 20,000+ TPS & 2–25 ms execution time on any sized orderbook with full network optimizations in place.
